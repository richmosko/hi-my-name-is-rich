So admittedly I'm a bit late to the AI party. When chatGPT came out in 2023 I viewed it rather skeptically and thought that its capabilities were a bit oversold. How useful could something be that was simply predicting next words into likely sentences? Apparently pretty useful.

Most of my friends have been using chatGPT over the past year or two, and I have to admit that some of the things it does are pretty impressive. While in Bali last summer, my girlfriend simply took a picture of a prescription drug given to me... and "Chat" read the label, translated it from Indonesian to English, summarized what the drug was typically used for, and told me what the likely dosage amount should be given my flu-like symptoms. It was amazing. And while at home reading through myChart lab data and imaging on my Dad's condition, Chat was able to fully diagnose what was going on, and give likely scenarios for treatment options. Again, simply amazing. I felt like this technology was (and is) going to change the world. I needed to catch up on it as expeditiously as possible.

So I find myself curious. And possibly a bit engrossed and consumed. I want to know all that there is about this technology: How it works. What it's useful for. How it might impact humanity. This is my plan of attack...

# Building
Nothing like getting your hands dirty to figure something out. So I ponied up for an Anthropic subscription and started using Claude for day to day things. It was novel... but not particularly useful? Aside from the occasional medical diagnosis. Or frankly, psychological diagnosis. In any case, I wasn't really using it much at first because Google was doing such a great job of embedding Gemini into all searches. Still is, really. But what I discovered was that AI (and Claude-Code specifically) was excellent at coding. And architecting software. So I came up with two projects to drive what people call "Vibe Coding":
#### Hi, My Name is Rich
That's this site! I am writing this after the fact... but this site is fully build with claude-code. I know nothing about frontend design except what I've picked up along the way. Nothing about React, or Vite, or Tradewinds, or CSS or any of that technical nonsense. And the deployment was done with a significant amount of expertise from claude telling me how to set up open source software on a virtual server. So this whole bad boy is self-designed **and** self-hosted. Pretty slick.
This is essentially a vibe-code trial run. It's nice to have a personal blog and web site. Nice to be able to document and track my progress on things. But not necessary. Really the use here is to chronicle my design progress as to what works and doesn't work with AI. Especially on the next project...
#### Personal Finance Dashboard
This is a more practical project. For years, I have done my own accounting in terms of tracking Transactions, Financial Portfolios, Budgeting, Tax Planning, and Asset Allocation. I create a report every month that tracks all of this. It's basically my only true job. Over the years I've written a ton of javascript in Google Sheets to query things and automate much of the flow... but it still takes me the better part of a day to complete, and granularity is literally in one-month snapshots. Not ideal. I have always wanted to improve this process and create a web based dashboard similar to the old personalcapital.com... but with features tailored to me. I would have never been able to do this in the past. That's an entire fintech platform!! crazy. But I think that I can do this with AI. Actually, as I am writing this months later... I know I can. By myself. With zero starting experience and almost no capital expenditures needed. A full-stack online application:
- Virtual Server hosted remotely
- Open Source Database
- OAuth Authentication
- Automated Docker Deployment
- Open Source Platform as a Service for managing service deployment, proxying, and SSL certificates
- Custom SQL architecture with row-level security and graphQL API
- User 2FA authentication
- Frontend infrastructure
- Frontend design

... as well as various scripts and functions to make the application work. It's a stretch goal, but I think it'd doable.
